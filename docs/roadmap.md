# Roadmap

## Fase 0 - Fundacao

Objetivo: projeto rodando localmente, com decisoes claras.

- API Kotlin/Spring Boot criada.
- Documentacao de produto.
- ADRs de arquitetura e privacidade.
- Endpoint de health.

## Fase 1 - Livro mensal

Objetivo: substituir a planilha no uso basico.

- SQLite local.
- Migrations.
- CRUD simples de lancamentos.
- Resumo mensal.
- Testes dos calculos.
- Exportacao CSV anonimizavel.

## Fase 2 - Performance e economia

Objetivo: entender se o mes esta saudavel.

- Meta mensal de investimento.
- Porcentagem investida no mes.
- Performance contra meta.
- Total anual de entradas.
- Total anual investido.
- Porcentagem anual investida.

## Fase 3 - Advisor de compra

Objetivo: responder se uma compra cabe.

- Simulacao a vista.
- Simulacao parcelada.
- Impacto no mes atual.
- Impacto nos meses futuros.
- Explicacao da decisao.

## Fase 4 - Interface

Objetivo: tornar o uso diario rapido.

- Tela de lancamento.
- Tela de resumo mensal.
- Tela de simulacao.
- Visual do termometro.

## Fase 5 - Investimentos

Objetivo: evoluir planejamento de investimento sem virar corretora.

- Metas por tipo de reserva.
- Historico de aportes.
- Projecao simples.
- Comparacao contra meta anual.

## Fase 6 - Sincronizacao opcional

Objetivo: abrir em outro PC sem perder privacidade.

Opcoes a avaliar:

- backup manual criptografado;
- pasta sincronizada pelo usuario;
- Firebase/Auth somente para app com login;
- provider relacional free tier somente se houver necessidade real.
